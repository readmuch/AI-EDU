# Mobile UX Redesign Design

## Goal

수강생이 모바일에서 교육 구조를 빠르게 이해하고, 수강 중 필요한 실습 예제와 프롬프트 템플릿을 쉽게 찾고 복사할 수 있는 서비스형 화면으로 개편한다.

## Current Problems

- 현재 `App.jsx`는 모든 섹션을 긴 단일 페이지로 나열한다. 모바일에서는 원하는 정보까지 스크롤 비용이 크다.
- `LecturePlanSection.jsx`가 많은 하위 정보를 한 번에 펼쳐 보여준다. 카드 안에 카드가 반복되어 모바일 가독성이 낮다.
- 첫 화면의 CTA가 3개로 나뉘어 있어 모바일 사용자가 우선 행동을 고르기 어렵다.
- 주요 자료인 실습 예제와 프롬프트 템플릿이 페이지 중하단에 있어 수강 중 재접근성이 낮다.
- 현재 한글 콘텐츠가 일부 깨진 상태라 디자인 개편 시 실제 한국어 문구 기준으로 정리해야 한다.

## Recommended Direction

`A+B 혼합` 구조로 개편한다.

- 하단 고정 탭을 도입해 모바일 이동성을 높인다.
- 홈 첫 화면은 3단계 학습 경로를 중심으로 구성한다.
- 홈 상단에 `실습 예제`와 `템플릿` 바로가기 버튼을 크게 배치한다.
- 긴 강의 상세는 기본 접힘 상태의 아코디언과 탭으로 나눠 스크롤 피로를 줄인다.

## Information Architecture

하단 탭은 5개로 제한한다.

1. 홈
2. 과정
3. 실습
4. 템플릿
5. FAQ

각 탭의 역할:

- `홈`: 핵심 소개, 3단계 학습 경로, 바로가기 CTA
- `과정`: 단계별 커리큘럼과 상세 강의 계획
- `실습`: 업무 유형별 실습 예제, 복사 가능한 예시
- `템플릿`: 프롬프트 템플릿, 카테고리 필터, 복사 기능
- `FAQ`: 수강 전 자주 묻는 질문과 문의 안내

데스크톱에서는 하단 탭 대신 상단 또는 좌측 보조 내비게이션으로 표현할 수 있지만, 정보 구조는 동일하게 유지한다.

## Mobile Screen Flow

### Home

- 상단에는 짧은 제목과 설명만 둔다.
- 기본 CTA는 하나만 강조한다: `과정 보기`.
- 보조 CTA는 2개로 제한한다: `실습 예제`, `템플릿 복사`.
- 3단계 로드맵 카드를 바로 보여준다.
- 각 단계 카드는 `단계명`, `대상`, `소요 시간`, `핵심 결과`만 노출한다.

### Course

- 1단계, 2단계, 3단계 선택은 가로 스크롤 세그먼트나 큰 탭으로 제공한다.
- 선택한 단계의 상세 정보는 다음 순서로 접힘/펼침 처리한다.
  - 개요
  - 학습 목표
  - 준비물
  - 일자별 시간표
  - 실습 자료
  - 평가 기준
- 모바일 기본 상태에서는 `개요`, `학습 목표`, `일자별 시간표`만 펼치고 나머지는 접는다.

### Practice

- 업무 유형별 카드로 구성한다.
- 카드 제목은 `메일 작성`, `회의록 요약`, `보고서 작성`, `데이터 분석`처럼 실제 업무명으로 쓴다.
- 각 실습 카드는 접힘 상태에서 `상황`, `목표`, `복사 버튼`만 보인다.
- 펼치면 `프롬프트`, `예상 결과`, `활용 팁`을 볼 수 있다.

### Templates

- 상단에 카테고리 필터를 둔다.
- 템플릿 본문은 `pre` 영역을 유지하되 모바일에서 줄바꿈을 강제해 가로 스크롤을 최소화한다.
- 복사 버튼은 카드 상단과 하단 중 하나로 고정한다. 모바일에서는 하단 고정 버튼보다 카드 내부 버튼을 우선한다.

### FAQ

- `details` 기반 아코디언은 유지한다.
- 질문은 짧게 다듬고, 답변은 2-4문장 단위로 줄인다.

## Visual Design Rules

- 모바일 본문 기본 크기는 16px 이상으로 한다.
- 작은 보조 텍스트도 13px 미만으로 내려가지 않게 한다.
- 카드 radius는 8px 수준으로 낮춰 과한 둥근 카드 느낌을 줄인다.
- 섹션 전체를 카드로 감싸는 패턴을 줄이고, 반복 항목에만 카드를 사용한다.
- 한 화면에 보이는 강조색은 2개 이하로 제한한다.
- 보라/파랑 단일 팔레트에 치우치지 않게 `indigo`, `teal`, `amber`, `slate`를 역할별로 나눈다.
- 버튼 높이는 최소 44px로 맞춘다.
- 하단 탭은 아이콘과 짧은 라벨을 함께 제공한다.

## Component Architecture

새 구조는 다음 단위로 나눈다.

- `MobileShell`: 전체 레이아웃, 하단 탭, 현재 탭 상태 관리
- `BottomNavigation`: 모바일 하단 탭
- `HomeView`: 첫 화면과 3단계 학습 경로
- `CourseView`: 단계별 과정 상세
- `PracticeView`: 실습 예제 목록과 상세
- `TemplateView`: 템플릿 필터와 복사 UI
- `FaqView`: FAQ 목록
- `Accordion`: 긴 콘텐츠 접힘/펼침 공통 컴포넌트
- `CopyButton`: 복사 상태를 재사용하는 공통 버튼

기존 섹션 컴포넌트는 바로 삭제하지 않고, 데이터를 검증하면서 새 View 컴포넌트로 기능을 옮긴다.

## Data Strategy

- 기존 `courseData.js`와 `lectureDetailData.js`의 구조는 최대한 유지한다.
- 화면에서 필요한 요약 필드가 부족하면 별도 `mobileSummary` 필드를 추가한다.
- 깨진 한글 문자열은 UX 개편과 함께 정상 한국어로 복구한다.
- 복구가 어려운 항목은 동일 의미의 짧은 문구로 다시 작성한다.

## Accessibility

- 하단 탭은 `button` 요소와 `aria-current`를 사용한다.
- 탭 전환 후 화면 상단 제목으로 포커스를 옮기는 동작을 검토한다.
- 모든 복사 버튼은 복사 완료 상태를 텍스트로 알려준다.
- 색상만으로 선택 상태를 구분하지 않고, 굵기/테두리/라벨도 함께 사용한다.

## Testing And Verification

필수 확인:

- `npm run lint`
- `npm run build`
- 모바일 375px, 390px, 430px 폭에서 화면 확인
- 데스크톱 1280px 폭에서 화면 확인
- 하단 탭 전환 동작 확인
- 복사 버튼 동작 확인
- 긴 텍스트가 버튼/카드 밖으로 넘치지 않는지 확인
- Lighthouse 또는 브라우저 개발자 도구로 접근성 기본 항목 확인

## Out Of Scope

- 회원가입, 로그인, 진도 저장
- 서버/API 추가
- 학습 완료 체크 데이터 저장
- 결제/신청 폼
- 외부 CMS 연동

## Responsive Desktop Strategy

반응형 단일 앱 구조로 구현한다.

- 모바일은 하단 고정 탭을 사용한다.
- 데스크톱은 같은 탭 정보 구조를 상단 탭 또는 좌측 내비게이션으로 표현한다.
- 모바일과 데스크톱은 같은 View 컴포넌트와 같은 데이터를 공유한다.
- 화면 크기에 따라 내비게이션 위치와 콘텐츠 밀도만 바꾸고, 별도 랜딩 페이지 구조를 유지하지 않는다.
- 데스크톱에서는 `Home`, `Course`, `Practice`, `Templates`, `FAQ`를 넓은 화면에 맞게 2열 레이아웃 일부로 확장할 수 있다.

이 결정은 유지보수 비용을 낮추고, 모바일에서 정리한 정보 구조가 데스크톱에서도 일관되게 보이도록 하기 위한 것이다.
